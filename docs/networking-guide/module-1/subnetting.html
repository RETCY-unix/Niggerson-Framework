Subnetting

Overview

Subnetting is the practice of dividing a single network into smaller, more manageable network segments called subnets. This fundamental networking skill enables efficient address allocation, improved security through network segmentation, reduced broadcast domains, and optimized routing efficiency.

Understanding subnetting requires proficiency in binary arithmetic and a thorough understanding of IP address structure. This section will cover the mathematical foundations, practical calculation methods, and real-world application of subnetting principles.

Subnet
A logical subdivision of an IP network. Subnetting partitions a network into smaller network segments, each operating as an independent IP network while remaining part of the larger network infrastructure.

Purpose of Subnetting

Subnetting Benefits
  • Broadcast Domain Reduction: Limits broadcast traffic to smaller segments
  • Security: Enables network segmentation and access control
  • Address Efficiency: Allocates appropriate-sized networks
  • Routing Optimization: Reduces routing table size through summarization
  • Administrative Control: Simplifies network management
  • Performance: Reduces unnecessary traffic on network segments

Subnet Mask Fundamentals

A subnet mask is a 32-bit number that divides an IP address into network and host portions. The mask consists of consecutive 1 bits (network portion) followed by consecutive 0 bits (host portion). The boundary between 1s and 0s determines the network size.

SUBNET MASK STRUCTURE: Valid subnet mask: Consecutive 1s, then consecutive 0s VALID: 11111111.11111111.11111111.00000000 = 255.255.255.0 = /24 11111111.11111111.11111111.11000000 = 255.255.255.192 = /26 11111111.11111111.11110000.00000000 = 255.255.240.0 = /20 INVALID: 11111111.11111111.10101010.00000000 (non-consecutive 1s) 11111111.11111111.00000000.11111111 (1s after 0s) SUBNET MASK APPLICATION: IP Address: 192.168.10.130 11000000.10101000.00001010.10000010 Subnet Mask: 255.255.255.192 (/26) 11111111.11111111.11111111.11000000 NNNNNNN.NNNNNNNN.NNNNNNNN.NN------ (N = Network bits, - = Host bits) AND Operation: IP: 11000000.10101000.00001010.10000010 Mask: 11111111.11111111.11111111.11000000 ========================================= Network ID: 11000000.10101000.00001010.10000000 = 192.168.10.128 Host portion: .10 (last 6 bits: 000010 = 2)

CIDR Notation

Classless Inter-Domain Routing (CIDR) notation provides a compact way to express both an IP address and its subnet mask. The notation appends a forward slash and the number of network bits to the IP address.

CIDR TO SUBNET MASK CONVERSION: CIDR Binary Mask Decimal Mask ==== ========================================== =============== /8 11111111.00000000.00000000.00000000 255.0.0.0 /16 11111111.11111111.00000000.00000000 255.255.0.0 /24 11111111.11111111.11111111.00000000 255.255.255.0 /25 11111111.11111111.11111111.10000000 255.255.255.128 /26 11111111.11111111.11111111.11000000 255.255.255.192 /27 11111111.11111111.11111111.11100000 255.255.255.224 /28 11111111.11111111.11111111.11110000 255.255.255.240 /29 11111111.11111111.11111111.11111000 255.255.255.248 /30 11111111.11111111.11111111.11111100 255.255.255.252 /31 11111111.11111111.11111111.11111110 255.255.255.254 /32 11111111.11111111.11111111.11111111 255.255.255.255

Key Formulas
Number of Subnets = 2^s
  Where s = number of subnet bits borrowed

Number of Hosts = 2^h - 2
  Where h = number of host bits remaining
  Subtract 2 for network address and broadcast address

Block Size = 256 - Subnet Mask Value
  Or: Block Size = 2^h (host bits)

Example: /26 (255.255.255.192)
  Subnet bits in octet 4: 2 (11000000)
  Host bits: 6 (00111111)
  Subnets per Class C: 2^2 = 4
  Hosts per subnet: 2^6 - 2 = 62
  Block size: 256 - 192 = 64

Subnetting Procedure

Example 1: Determine Network Information

Given IP address 172.16.45.200/21, determine the network address, broadcast address, and valid host range.

STEP 1: Identify the octet boundary /21 = 21 network bits First 16 bits: First two octets (172.16) Remaining 5 bits: In third octet Third octet mask value: 21 - 16 = 5 network bits in octet 3 Binary: 11111000 = 248 Subnet mask: 255.255.248.0 STEP 2: Calculate block size Block size = 256 - 248 = 8 Subnets in third octet: 0, 8, 16, 24, 32, 40, 48, ... STEP 3: Find the network containing 172.16.45.200 45 / 8 = 5.625 (integer part = 5) 5 * 8 = 40 (network starts at 40) Next network = 48 Network address: 172.16.40.0/21 Broadcast: 172.16.47.255 (next network - 1) STEP 4: Determine host range First host: 172.16.40.1 Last host: 172.16.47.254 Broadcast: 172.16.47.255 SUMMARY: +-----------------------+-----------------------+ | Network Address | 172.16.40.0/21 | | Subnet Mask | 255.255.248.0 | | First Usable Host | 172.16.40.1 | | Last Usable Host | 172.16.47.254 | | Broadcast Address | 172.16.47.255 | | Total Hosts | 2046 | +-----------------------+-----------------------+

Example 2: Subnet a Network

Divide 10.0.0.0/8 to create subnets with at least 500 hosts each.

STEP 1: Determine required host bits Need 500+ hosts per subnet 2^9 = 512 (9 host bits needed) 512 - 2 = 510 usable hosts (meets requirement) STEP 2: Calculate new prefix Host bits = 9 Network bits = 32 - 9 = 23 New prefix: /23 STEP 3: Determine subnet mask /23 = 23 network bits Mask: 11111111.11111111.11111110.00000000 Mask: 255.255.254.0 STEP 4: List subnets Block size in 3rd octet: 256 - 254 = 2 Subnet 0: 10.0.0.0/23 (10.0.0.1 - 10.0.1.254) Subnet 1: 10.0.2.0/23 (10.0.2.1 - 10.0.3.254) Subnet 2: 10.0.4.0/23 (10.0.4.1 - 10.0.5.254) Subnet 3: 10.0.6.0/23 (10.0.6.1 - 10.0.7.254) ... Subnet 127: 10.0.254.0/23 (10.0.254.1 - 10.0.255.254)

Variable Length Subnet Masking (VLSM)

VLSM allows subnets of different sizes within the same network, enabling optimal address allocation based on actual requirements. This technique prevents address waste that occurs with fixed-size subnetting.

VLSM EXAMPLE: Network to subnet: 192.168.1.0/24 Requirements: - LAN A: 60 hosts - LAN B: 25 hosts - LAN C: 12 hosts - WAN Link 1: 2 hosts - WAN Link 2: 2 hosts ALLOCATION (largest to smallest): 1. LAN A (60 hosts): Need /26 (62 usable) Network: 192.168.1.0/26 Range: 192.168.1.1 - 192.168.1.62 Bcast: 192.168.1.63 2. LAN B (25 hosts): Need /27 (30 usable) Network: 192.168.1.64/27 Range: 192.168.1.65 - 192.168.1.94 Bcast: 192.168.1.95 3. LAN C (12 hosts): Need /28 (14 usable) Network: 192.168.1.96/28 Range: 192.168.1.97 - 192.168.1.110 Bcast: 192.168.1.111 4. WAN Link 1 (2 hosts): Need /30 (2 usable) Network: 192.168.1.112/30 Range: 192.168.1.113 - 192.168.1.114 Bcast: 192.168.1.115 5. WAN Link 2 (2 hosts): Need /30 (2 usable) Network: 192.168.1.116/30 Range: 192.168.1.117 - 192.168.1.118 Bcast: 192.168.1.119 Remaining: 192.168.1.120 - 192.168.1.255 (available for future use) ADDRESS MAP: 0 64 96 112 120 192 256 |---A----|---B----|--C-|WW|--AVAILABLE--| /26 /27 /28 /30

Supernetting and Route Aggregation

Supernetting (also called route aggregation or summarization) combines multiple contiguous networks into a single larger network prefix. This reduces routing table size and improves routing efficiency.

SUPERNETTING EXAMPLE: Four /24 networks to aggregate: 192.168.0.0/24 192.168.1.0/24 192.168.2.0/24 192.168.3.0/24 Binary analysis of third octet: 0 = 00000000 1 = 00000001 2 = 00000010 3 = 00000011 ^^^^^^-- These 6 bits vary ^^ The last 2 bits differ Common bits: 000000xx (first 6 bits are common) Borrow 2 bits from network portion Result: 192.168.0.0/22 Verification: /22 mask = 255.255.252.0 Block size = 256 - 252 = 4 Network covers: 192.168.0.0 - 192.168.3.255 ROUTE TABLE COMPARISON: Without Aggregation: With Aggregation: +------------------+ +------------------+ | 192.168.0.0/24 | | 192.168.0.0/22 | | 192.168.1.0/24 | ==> +------------------+ | 192.168.2.0/24 | | 192.168.3.0/24 | (4 entries become 1) +------------------+
