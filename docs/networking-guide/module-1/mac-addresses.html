Media Access Control Addresses

Overview

A Media Access Control (MAC) address is a hardware identifier assigned to network interface controllers (NICs) for communications at the Data Link Layer (Layer 2) of the network stack. MAC addresses are sometimes referred to as physical addresses, hardware addresses, or burned-in addresses (BIA), reflecting their permanent association with network hardware.

Unlike IP addresses which are logical and can change based on network configuration, MAC addresses are intended to be globally unique and permanently assigned to hardware by manufacturers. They enable communication within a local network segment before packets are routed to other networks.

MAC Address
A 48-bit (6-byte) unique identifier assigned to a network interface controller. MAC addresses operate at Layer 2 and are used for frame delivery within a local network segment.

MAC Address Structure

A MAC address consists of 48 bits (6 bytes), typically represented as 12 hexadecimal digits. The address is divided into two halves: the first 3 bytes identify the manufacturer (OUI), and the last 3 bytes identify the specific device.

MAC ADDRESS FORMAT: 48 bits = 6 bytes = 12 hexadecimal characters Notation Formats: +------------------------------------------+ | Colon-separated: 00:1A:2B:3C:4D:5E | | Hyphen-separated: 00-1A-2B-3C-4D-5E | | Dot-separated: 001A.2B3C.4D5E | | No separator: 001A2B3C4D5E | +------------------------------------------+ Structure: +------------------------+------------------------+ | OUI | NIC | | (Organizational | (Network Interface | | Unique Identifier) | Controller Specific) | +------------------------+------------------------+ | 3 bytes (24 bits) | 3 bytes (24 bits) | | 00:1A:2B | 3C:4D:5E | +------------------------+------------------------+ | Manufacturer ID | Device Serial | +------------------------+------------------------+

OUI (Organizationally Unique Identifier)

The first three bytes (24 bits) of a MAC address constitute the OUI, which identifies the manufacturer or vendor of the network interface. OUIs are assigned by the IEEE Registration Authority, ensuring global uniqueness.

Common OUI Assignments

Special Bits in MAC Addresses

The first byte of a MAC address contains two special bits that indicate the address type:

FIRST BYTE BIT BREAKDOWN: Byte 0 (First byte): XX:XX:XX:XX:XX:XX ^^ Bit 0 (LSB) - Unicast/Multicast flag: 0 = Unicast (single destination) 1 = Multicast (group destination) Bit 1 - Universal/Local flag: 0 = Universally Administered (OUI-based, vendor assigned) 1 = Locally Administered (manually assigned) Example: 00:1A:2B:3C:4D:5E 00 = 0000 0000 ||| ||+-- Bit 0 = 0: Unicast |+--- Bit 1 = 0: Universally Administered Special Addresses: FF:FF:FF:FF:FF:FF = Broadcast (all stations) 01:XX:XX:XX:XX:XX = Multicast group
Address Type Bit Pattern Description
Unicast Bit 0 = 0 Frame destined for single interface
Multicast Bit 0 = 1 Frame destined for group of interfaces
Broadcast FF:FF:FF:FF:FF:FF Frame destined for all interfaces on segment
Universally Administered Bit 1 = 0 Vendor-assigned, globally unique
Locally Administered Bit 1 = 1 Manually assigned, local scope only

Ethernet Frame Structure

MAC addresses are carried within Ethernet frames, the fundamental data unit at Layer 2. Every frame contains source and destination MAC addresses that determine how switches forward the frame.

ETHERNET FRAME FORMAT (IEEE 802.3): +----------+------------------+------------------+------+------+----------+-----+ | PREAMBLE | DEST MAC ADDRESS | SRC MAC ADDRESS | TYPE | DATA | PADDING | FCS | +----------+------------------+------------------+------+------+----------+-----+ | 7 bytes | 6 bytes | 6 bytes | 2B | 46- | 0-46 | 4B | | | | | | 1500 | bytes | | | | | | | bytes| | | +----------+------------------+------------------+------+------+----------+-----+ | 10101010 | FF:FF:FF:FF:FF:FF| 00:1A:2B:3C:4D:5E| 0800 | | |CRC32| |repeating | (broadcast) | (source) |(IPv4)| | | | +----------+------------------+------------------+------+------+----------+-----+ PREAMBLE: Synchronization pattern (10101010... 10101011) DEST MAC: Destination hardware address SRC MAC: Source hardware address TYPE/LENGTH: EtherType (0x0800=IPv4, 0x0806=ARP, 0x86DD=IPv6) DATA: Payload from upper layers (minimum 46 bytes) PADDING: Added if data less than 46 bytes FCS: Frame Check Sequence (CRC-32 for error detection) Minimum Frame: 64 bytes Maximum Frame: 1518 bytes (1522 with VLAN tag) MTU: 1500 bytes (maximum payload)

Address Resolution Protocol (ARP)

ARP is the protocol that maps IP addresses to MAC addresses on a local network segment. When a device needs to communicate with another device on the same network, it must know the destination MAC address to construct the Ethernet frame. ARP provides this mapping.

ARP Operation
ARP uses broadcast requests to discover the MAC address associated with a known IP address. The device with the matching IP responds with its MAC address, which is then cached for future use.
ARP RESOLUTION PROCESS: Host A (192.168.1.10) wants to communicate with Host B (192.168.1.20) Host A knows IP but not MAC address of Host B STEP 1: ARP Request (Broadcast) +-------------+ +-------------+ | HOST A | | HOST B | | 192.168.1.10| | 192.168.1.20| | AA:AA:AA:.. | | BB:BB:BB:.. | +-------------+ +-------------+ | | | ARP REQUEST (Broadcast to FF:FF:FF:FF:FF:FF) | | "Who has 192.168.1.20? Tell 192.168.1.10" | |------------------------------------------------------->| | (All hosts receive) | | | STEP 2: ARP Reply (Unicast) | | | ARP REPLY (Unicast to AA:AA:AA:AA:AA:AA) | | "192.168.1.20 is at BB:BB:BB:BB:BB:BB" | |<-------------------------------------------------------| | | STEP 3: Cache Entry Created Host A caches: 192.168.1.20 -> BB:BB:BB:BB:BB:BB STEP 4: Communication Proceeds | | | Ethernet Frame (Unicast to BB:BB:BB:...) | | Contains IP packet destined for 192.168.1.20 | |------------------------------------------------------->|

ARP Cache

Devices maintain an ARP cache (or ARP table) storing recent IP-to-MAC mappings. Cache entries have a timeout (typically 2-20 minutes) after which they expire and must be refreshed through new ARP requests.

ARP Cache Commands
Windows:
  arp -a                    # Display ARP cache
  arp -d *                  # Clear ARP cache
  arp -s 192.168.1.50 00-1A-2B-3C-4D-5E  # Static entry

Linux:
  arp -n                    # Display ARP cache
  ip neigh show             # Modern alternative
  ip neigh flush all        # Clear ARP cache
  arp -s 192.168.1.50 00:1A:2B:3C:4D:5E  # Static entry

Switch MAC Address Tables

Network switches maintain a MAC address table (also called CAM table - Content Addressable Memory) that maps MAC addresses to physical switch ports. This table enables switches to efficiently forward frames only to the appropriate destination port rather than flooding all ports.

SWITCH MAC ADDRESS LEARNING: Initial State: Empty MAC table [PC-A] [PC-B] [PC-C] [PC-D] AA:AA BB:BB CC:CC DD:DD | | | | Port1 Port2 Port3 Port4 +-------+-------+-------+ | | | [SWITCH] | | | +-----------------------+ Step 1: PC-A sends frame to PC-C Switch learns: Port1 -> AA:AA (source of frame) Switch does not know Port for CC:CC Switch floods frame to all ports except source Step 2: PC-C responds to PC-A Switch learns: Port3 -> CC:CC Switch knows AA:AA is on Port1 Switch forwards ONLY to Port1 (no flooding) After learning: +------------------+ | MAC Address Table| +-------+----------+ | Port | MAC | +-------+----------+ | 1 | AA:AA | | 2 | BB:BB | | 3 | CC:CC | | 4 | DD:DD | +-------+----------+
Switch MAC Table Commands (Cisco)
show mac address-table           # Display full MAC table
show mac address-table dynamic   # Show dynamically learned entries
show mac address-table interface Gi0/1  # Show MACs on specific port
clear mac address-table dynamic  # Clear dynamic entries

MAC Address Security Considerations

MAC Spoofing

MAC spoofing is the practice of changing a device's MAC address to impersonate another device. Since MAC addresses are processed in software, they can be modified despite being "burned in" to hardware.

MAC Spoofing Commands
Windows (via Registry or Device Manager):
  - Network adapter properties
  - Advanced > Network Address

Linux:
  ifconfig eth0 down
  ifconfig eth0 hw ether XX:XX:XX:XX:XX:XX
  ifconfig eth0 up
  
  # Or using ip command:
  ip link set eth0 down
  ip link set eth0 address XX:XX:XX:XX:XX:XX
  ip link set eth0 up

macOS:
  sudo ifconfig en0 ether XX:XX:XX:XX:XX:XX

ARP Attacks

Attack Description Impact
ARP Spoofing Sending fake ARP replies to poison cache Man-in-the-middle, traffic interception
ARP Cache Poisoning Corrupting ARP cache with false entries Traffic redirection, DoS
CAM Table Overflow Flooding switch with fake MACs Switch fails open, broadcasts all traffic
MAC Flooding Overwhelming switch MAC table capacity Switch becomes hub, enables sniffing

Defensive Measures

Layer 2 Security Controls
  • Port Security: Limit MACs per port, sticky MAC learning
  • Dynamic ARP Inspection (DAI): Validates ARP packets against DHCP snooping database
  • DHCP Snooping: Builds trusted IP-MAC bindings
  • 802.1X: Port-based network access control
  • Private VLANs: Isolate hosts within same VLAN
  • Static ARP Entries: For critical infrastructure

This section covered MAC addresses and Layer 2 communication:

  • Special address types: unicast, multicast, broadcast

Key Takeaways

  • ARP bridges Layer 2 and Layer 3 by resolving IP to MAC
  • Switches learn MAC addresses dynamically from frame sources
  • MAC addresses can be spoofed despite being hardware-based
  • Layer 2 security requires specific controls (port security, DAI)
