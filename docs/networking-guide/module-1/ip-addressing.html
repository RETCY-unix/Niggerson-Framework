Internet Protocol Addressing

Overview

Internet Protocol (IP) addressing constitutes the fundamental mechanism by which devices are identified and located within IP-based networks. Every device participating in network communication must possess a unique IP address that distinguishes it from all other devices within the same network scope. This addressing system forms the backbone of all internetwork communication, from local area networks to the global Internet.

IP addresses serve two primary functions: host identification and location addressing. Unlike physical MAC addresses which are permanently assigned to hardware, IP addresses are logically assigned and can be changed based on network configuration. This logical assignment enables hierarchical network design, efficient routing, and flexible network reorganization.

IP Address
A numerical label assigned to each device connected to a computer network that uses the Internet Protocol for communication. It serves both as an identifier for the interface and as a locator address for routing purposes.

IPv4 Addressing

Internet Protocol version 4 (IPv4) has served as the primary addressing system since the early development of internetworking. An IPv4 address consists of 32 bits, typically represented as four decimal octets separated by periods (dotted-decimal notation). Each octet represents 8 bits, with values ranging from 0 to 255.

IPv4 Address Structure
IPv4 Address Length: 32 bits (4 bytes)
Total Possible Addresses: 2^32 = 4,294,967,296

Binary Format:     11000000.10101000.00000001.00000001
Decimal Format:    192.168.1.1

Octet Breakdown:
+-----------+-----------+-----------+-----------+
| Octet 1   | Octet 2   | Octet 3   | Octet 4   |
| (8 bits)  | (8 bits)  | (8 bits)  | (8 bits)  |
+-----------+-----------+-----------+-----------+
| 0-255     | 0-255     | 0-255     | 0-255     |
+-----------+-----------+-----------+-----------+

Binary to Decimal Conversion

Understanding binary-to-decimal conversion is essential for network operations. Each bit position in an octet represents a power of 2, from 2^7 (128) to 2^0 (1). Converting binary to decimal involves summing the values of all positions where the bit is set to 1.

BINARY POSITIONAL VALUES (8-bit octet): Position: 7 6 5 4 3 2 1 0 Value: 128 64 32 16 8 4 2 1 === == == == = = = = Example: Convert 11000000 to decimal Position: 7 6 5 4 3 2 1 0 Bit: 1 1 0 0 0 0 0 0 Value: 128 64 -- -- -- -- -- -- Sum: 128 + 64 = 192 Therefore: 11000000 (binary) = 192 (decimal)

IPv4 Address Classes

The original IPv4 addressing scheme defined five address classes (A through E) based on the leading bits of the first octet. This classful addressing system allocated addresses in fixed blocks, which proved inefficient for many organizations.

Private IP Address Ranges

RFC 1918 defines three address ranges reserved for private network use. These addresses are not routable on the public Internet and can be freely used within private networks. Network Address Translation (NAT) enables devices with private addresses to communicate with public Internet resources.

Subnet Masks

A subnet mask is a 32-bit number that divides an IP address into network and host portions. The mask uses consecutive 1s to identify network bits and consecutive 0s to identify host bits. The subnet mask determines which portion of an IP address identifies the network and which portion identifies the specific host within that network.

SUBNET MASK OPERATION: IP Address: 192.168.1.100 11000000.10101000.00000001.01100100 Subnet Mask: 255.255.255.0 11111111.11111111.11111111.00000000 ================================ Network ID: 192.168.1.0 (AND operation result) 11000000.10101000.00000001.00000000 NETWORK PORTION | HOST ========================= | ==== Host ID: .100 00000000.00000000.00000000.01100100

The AND operation between an IP address and its subnet mask yields the network address. All hosts sharing the same network address belong to the same IP network and can communicate directly without routing.

CIDR Notation

Classless Inter-Domain Routing (CIDR) provides a more flexible method for specifying network addresses than classful addressing. CIDR notation appends a forward slash and the number of network bits to the IP address.

CIDR Subnet Mask Total Hosts Usable Hosts
/8 255.0.0.0 16,777,216 16,777,214
/16 255.255.0.0 65,536 65,534
/24 255.255.255.0 256 254
/25 255.255.255.128 128 126
/26 255.255.255.192 64 62
/27 255.255.255.224 32 30
/28 255.255.255.240 16 14
/29 255.255.255.248 8 6
/30 255.255.255.252 4 2
/31 255.255.255.254 2 2 (point-to-point)
/32 255.255.255.255 1 1 (host route)
Usable Host Calculation
Usable Hosts = 2^(32 - CIDR prefix) - 2

The subtraction of 2 accounts for:
  1. Network Address (all host bits = 0)
  2. Broadcast Address (all host bits = 1)

Example: /24 network
  Host bits = 32 - 24 = 8
  Total addresses = 2^8 = 256
  Usable hosts = 256 - 2 = 254

IPv6 Addressing

Internet Protocol version 6 (IPv6) was developed to address the exhaustion of IPv4 address space and to provide improvements in routing, security, and auto-configuration capabilities. An IPv6 address consists of 128 bits, providing a vastly larger address space than IPv4.

IPv6 Address Space
IPv6 Address Length: 128 bits (16 bytes)
Total Possible Addresses: 2^128 = 340,282,366,920,938,463,463,374,607,431,768,211,456

Comparison:
  IPv4: 4.3 x 10^9 addresses
  IPv6: 3.4 x 10^38 addresses

That is approximately 340 undecillion addresses, or enough to assign:
  - 100 addresses per atom on Earth's surface
  - 4.8 x 10^28 addresses per human on Earth

IPv6 Address Format

IPv6 addresses are represented as eight groups of four hexadecimal digits, separated by colons. Each group represents 16 bits (2 bytes) of the address.

IPv6 ADDRESS FORMAT: Full Format: 2001:0db8:85a3:0000:0000:8a2e:0370:7334 Breakdown: +------+------+------+------+------+------+------+------+ | 2001 | 0db8 | 85a3 | 0000 | 0000 | 8a2e | 0370 | 7334 | +------+------+------+------+------+------+------+------+ 16b 16b 16b 16b 16b 16b 16b 16b Total: 8 groups x 16 bits = 128 bits

IPv6 Address Simplification Rules

IPv6 addresses can be simplified using two rules to reduce representation length:

Simplification Rules
RULE 1: Leading zeros within each group can be omitted

  2001:0db8:0000:0042:0000:0000:0000:0001
  2001: db8:   0:  42:   0:   0:   0:   1

RULE 2: Consecutive groups of all zeros can be replaced with ::
        (This can only be done ONCE per address)

  2001:db8:0:42:0:0:0:1
  2001:db8:0:42::1

Full simplification example:
  Original:   2001:0db8:0000:0000:0000:0000:0000:0001
  After R1:   2001:db8:0:0:0:0:0:1
  After R2:   2001:db8::1

IPv6 Address Types

Type Prefix Description
Global Unicast 2000::/3 Publicly routable addresses (equivalent to public IPv4)
Link-Local fe80::/10 Auto-configured addresses for local segment only
Unique Local fc00::/7 Private addresses not routed on Internet (similar to RFC 1918)
Multicast ff00::/8 One-to-many communication
Loopback ::1/128 Local host loopback
Unspecified ::/128 Absence of address

IP Address Assignment Methods

Static Assignment

Static IP addressing involves manually configuring an IP address on each device. This method provides absolute control over addressing but requires manual administration and documentation. Static addressing is typically used for servers, network infrastructure, and devices requiring consistent addressing.

Dynamic Assignment (DHCP)

Dynamic Host Configuration Protocol (DHCP) automates IP address assignment from a configured pool. A DHCP server leases addresses to clients for a specified duration, reducing administrative overhead and enabling efficient address utilization.

IPv6 Auto-Configuration

IPv6 supports Stateless Address Autoconfiguration (SLAAC), which enables devices to configure their own addresses without a DHCP server. The device generates its interface ID (typically from its MAC address) and combines it with the network prefix advertised by routers.

  • Subnet masks and network/host portion identification

Key Takeaways

  • IPv4 exhaustion drove development of IPv6 with vastly expanded address space
  • Subnet masks determine network boundaries and enable efficient routing
  • CIDR enables flexible address allocation beyond classful boundaries
  • Private addressing conserves public address space through NAT
  • IPv6 simplifies address management through auto-configuration capabilities
